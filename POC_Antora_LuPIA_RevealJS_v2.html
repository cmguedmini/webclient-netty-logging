Documentation centralisée, versionnée et maintenable

Pourquoi Antora ?

  • Génération de sites statiques
  • Sources Git
  • Versionning de la documentation
  • Modularité (Components / Modules)
  • Intégration CI/CD

Concepts fondamentaux

Component = domaine documentaire
Module = sous-ensemble du composant
Page = fichier AsciiDoc
Playbook = agrégation des sources

  1. Créer une page .adoc
  2. Mettre à jour nav.adoc
  3. Ajouter une image ou un diagramme
  4. Commit Git
  5. Build Antora
